# M5-6f UI 一致性驗收

## 一、驗收結論

M5-6f 通過。14 個 UI 場景均使用 1280×720 邏輯尺寸、共用 `textStyle()` 與既有 UI 元件；未發現場景直接依實體畫布尺寸排版、繞過共用文字樣式、或重新引入已知模糊渲染設定。桌面瀏覽器抽驗畫面可讀、選中態與面板樣式一致，console 無錯誤。

2× 超取樣在行動 GPU 的幀數、窄螢幕安全區與首次載入時間不屬於本次視覺一致性修正，依規劃轉入 M5-8 網頁版架構優化。

## 二、自動檢查

執行：

```powershell
node tools/validate-ui-consistency.mjs
```

結果：

- UI 場景：14 個。
- `scene.add.text` 節點：131 個，全部使用 `textStyle()`。
- 字級範圍：10～56 px，未低於最低可讀門檻 10 px。
- 14 場景全部明確採用 `BASE_W`／`BASE_H`，未使用 `this.scale.width`／`this.scale.height` 排版。
- 未命中場景內直接 `fontFamily`／`fontSize`、直接 `new Text`、`mipmapFilter` 或 `antialiasGL`。
- `ui.ts` 維持 1280×720 設計尺寸與 `TEXT_RES = 4`。

## 四、瀏覽器抽驗

- 環境：本機 Vite 5173、1280×720 CSS viewport、Phaser canvas 2560×1440（2× 超取樣）。
- 抽驗：標題、港町、交易所、資訊選單、夥伴資訊、設定覆蓋層、存檔位置。
- 互動：港町選單、交易所離開、資訊頁籤、設定開啟與返回、存檔頁進入／返回皆可操作。
- Console：0 筆 error。
- 本次未改存檔格式，也未覆寫空白存檔格。

## 五、M5-8 交接基線

- 本機重載後先出現純色載入畫面，約十多秒後才完整顯示標題與背景；此數字只作本機開發基線，不等同正式行動網路測量。
- 當前啟動流程仍 eager 載入大量素材；M5-8 應量測 production build 的傳輸量、首次可玩時間與各場景載入峰值，再決定 lazy load、WebP／縮圖與快取策略。
- 行動裝置要另驗觸控命中、直橫向、瀏海安全區及 2× 超取樣幀數；不可用桌面 1280×720 結果代替。
