# 夥伴招募任務缺失清單與交接待辦

> 目的：記錄 25 位夥伴目前的實際招募機制、與《遊戲建構書》§5-7 的差異，以及後續交由小航接手的實作順序與驗收標準。
>
> 本檔是缺失與交接清單，不取代《遊戲建構書》。若實作方向需要偏離建構書，必須先由老闆確認。

---

## 一、清查結論

目前夥伴系統已具備：

- 25 位夥伴資料、港口、費用、職位、能力值與圖鑑。
- 章節、主角線、資金、探索點等被動招募門檻。
- 12 位 ★4～5 夥伴的招募完成對話。
- 顏思齊、鄭芝龍兩人的客座離隊資料。

但「專屬任務」尚未真正完成。目前 `questStages` 只是每次打開畫面時，重新檢查玩家當下是否符合條件；沒有任務接取、進行中、完成旗標、任務日誌，也沒有送貨、海戰、跑港、對話或採集等可操作流程。12 位高星夥伴的劇情只會在所有門檻達成後播放，因此屬於「招募完成演出」，不是任務過程。

| 類別 | 人數 | 現況 |
|---|---:|---|
| ★4～5 高星夥伴 | 12 | 有被動條件清單及招募完成對話，但沒有真正可接取的專屬任務 |
| 有條件的 ★3 夥伴 | 7 | 只有章節／資金／探索門檻，多數連招募劇情也沒有 |
| 直接付費加入 | 6 | 其中只有尤蘇夫符合建構書；另外 5 位原定小任務尚未實作 |

## 二、共通缺失

### P0：玩家看不到條件

- `showModal()` 高度固定，條件內容過長時會被裁切，底部又被「知道了」按鈕遮住。
- 畫面只顯示「查看條件」，沒有在夥伴卡片直接顯示下一個明確目標。
- 沒有捲動、分頁或展開式任務內容。
- 沒有在任務日誌、世界地圖或港口資訊中提示夥伴所在地與下一步。

### P0：截圖中的阿迪卡狀態異常

依目前正式資料，阿迪卡應位於萬丹、沒有前置條件、費用 400 兩。玩家持有 1175 兩時，按鈕應顯示「結識 400兩」，不應在巴達維亞顯示「查看條件」。需要實機重現並確認：

- MatesScene 收到的 `portId` 是否正確。
- 候選人列表與按鈕 callback 是否對應同一位夥伴。
- 切換港口、分頁、重建列表後是否殘留舊互動區。
- 瀏覽器是否仍執行舊版開發伺服器或快取內容。

### P1：沒有真正的任務狀態

- `questStages` 沒有 accepted／active／completed 狀態。
- 資金門檻是即時判定；玩家達標後花錢，畫面會重新變成未完成。
- 沒有永久記錄某一步曾經完成。
- 沒有任務失敗、限時過期、取消或重新接取規則。
- 沒有存檔遷移欄位；若新增任務進度，需將存檔由 v17 升版。

### P1：任務名稱與實際玩法不一致

- 「危機」「請託」「試探」「商帳」等名稱看似有事件，但實際只是章節或金錢門檻。
- 探索型任務只檢查既有探索點，沒有接任務後才出現的提示或劇情。
- 12 份高星劇情全部是條件達標後的入隊對話，沒有任務開場與中段。
- 聲望、友好度、累積貿易額、指定海戰、採集、送貨、跑港與對話任務尚未建立。

## 四、建議交接實作順序

### 第一階段：P0 UI 與錯位問題

- [ ] 將夥伴條件改為可捲動、分頁或自適應高度的專用視窗，不再用會裁字的通用短彈窗。
- [ ] 卡片直接顯示「下一步」；例如「下一步：前往首里城探索」。
- [ ] 條件視窗顯示夥伴所在港、主角線限制、章節、費用、已完成進度與尚未完成項目。
- [ ] 重現「巴達維亞畫面卻開出阿迪卡條件」；確認 portId、候選人、按鈕 callback 與重建列表生命週期。
- [ ] 永遠不能在目前主角線招募者，改為隱藏或明確標示「此路線無法招募」。

### 第二階段：P1 資料驅動任務框架

- [ ] 決定新增 `src/data/mate_quests.json`，或擴充 `mates.json`；任務內容不得寫死在場景程式。
- [ ] GameState 新增持久化夥伴任務進度，例如 `mateQuests[questId] = { status, stageIndex, counters }`。
- [ ] 存檔由 v17 升版並建立舊檔遷移，舊玩家不可失去已招募夥伴。
- [ ] 支援任務狀態：未發現、可接、進行中、可回報、完成、失敗／過期。
- [ ] 支援目標類型：對話、抵達港口、累積貿易額、交付貨物、海戰勝利、探索點、圖鑑解鎖、持有資金、主線章節。
- [ ] 完成狀態必須永久保存；資金花掉後不可讓已完成階段倒退。
- [ ] 將夥伴任務加入資訊頁／任務日誌，提供所在地與下一步提示。

### 第三階段：P1 任務內容落地

- [ ] 先做垂直範例：建議劉香（接任務→海上遭遇→擊敗→廣州招募），驗證海戰型流程。
- [ ] 再做鄭和（多港蒐集）、李旦（貿易額／送貨）、理加（探索／友好）三種不同任務模板。
- [ ] 完成其餘 8 位 ★4～5 任務，將現有 12 份招募對話拆成開場、中段或完成演出。
- [ ] 完成 7 位 ★3 區域任務。
- [ ] 完成 5 位 ★1～2 小任務；尤蘇夫維持直接雇用。

### 第四階段：P1 主線要角規則修正

- [ ] 顏思齊：林線自動同行、彼得線 1625 前限時任務、千代線不可。
- [ ] 濱田彌兵衛：千代線主線自動加入；確認其他主角專屬任務如何依建構書保留。
- [ ] 鄭芝龍：維持千代線客座並驗證第 8 章後離隊。
- [ ] 鄭成功：補客座離隊；林線與千代線使用不同登場文本／時間。
- [ ] 施琅：林線正常招募；彼得線短期雇用並設定離隊條件。
- [ ] 鄭經：補見習航海任務及架空年代說明。

### 第五階段：測試與文件同步

- [ ] 為 25 位夥伴建立資料完整性測試：港口、任務、探索點、貨物、圖鑑及劇情 id 均存在。
- [ ] 三主角分別跑一次所有可招募／不可招募路線。
- [ ] 測試章節窗口、客座離隊、資金變動、任務中途存讀檔與舊存檔遷移。
- [ ] 確認所有條件在 UI 都能看見，且國小五六年級玩家能理解下一步。
- [ ] 實作完成後同步更新《遊戲建構書》、`status.md`、`memory.md`、`log.md`。

## 五、建議驗收標準

1. 玩家第一次遇到有條件夥伴時，能在一個畫面內理解「在哪裡、要做什麼、目前做到哪裡」。
2. 每個標示為「專屬任務」的夥伴，至少有一次主動接取及一次明確完成／回報行為。
3. ★4～5 任務至少包含 2 個階段；★5 可使用 3～4 個階段。
4. 任務進度寫入存檔，不因資金或貨物後續變動而倒退。
5. 三條主角線的限定、客座、離隊與不可招募規則符合建構書。
6. 25 位夥伴都有可完成路徑，或清楚標示該主角線不可招募；不得出現永遠卡住但不解釋的按鈕。
7. 招募完成後正確扣除費用、加入船隊、指派職位、解鎖圖鑑並播放對應劇情。

## 六、程式與資料切入點

| 檔案 | 目前責任 | 後續工作 |
|---|---|---|
| `src/data/mates.json` | 25 位夥伴與被動條件 | 校正條件、路線、客座資料；避免把任務標題當成任務本體 |
| `src/data/story/mates_夥伴任務.md` | 12 位高星招募完成對話 | 拆分或擴寫為任務開場／中段／完成演出 |
| `src/state.ts` | 條件即時計算、招募、客座離隊、存檔 | 新增持久化任務狀態、目標計數、升版遷移與資料驗證 |
| `src/scenes/MatesScene.ts` | 顯示候選人、查看條件、啟動招募劇情 | 專用條件視窗、接受／回報任務、下一步提示、修正錯位問題 |
| `src/scenes/StoryScene.ts` | 播放達標後招募對話 | 支援任務開場／階段／完成劇情，不應每次播放完都直接招募 |
| `src/ui.ts` | 通用短彈窗 | 不宜直接承載長任務內容；若擴充需避免影響其他場景 |
| `src/scenes/InfoScene.ts` | 資訊與任務頁 | 增加夥伴任務追蹤及下一步提示 |

## 七、交接提醒

- 先修 UI 可見性，再做任務框架；否則新增任務後玩家仍看不到內容。
- 不建議直接為 25 人各寫一套硬編碼流程；先建立資料驅動的共用目標類型。
- 現有 12 份高星劇情可保留，但應重新定位為任務分段演出。
- 此工作會改變存檔結構，必須升版與測試 v17 舊存檔。
- 主線要角涉及建構書既定劇情，不可自行更改可招募路線或史實離隊時機。
