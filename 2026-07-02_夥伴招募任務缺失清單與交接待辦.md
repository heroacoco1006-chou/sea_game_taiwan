# 夥伴招募任務缺失清單與交接待辦

> 目的：記錄 25 位夥伴目前的實際招募機制、與《遊戲建構書》§5-7 的差異，以及後續交由小航接手的實作順序與驗收標準。
>
> 本檔是缺失與交接清單，不取代《遊戲建構書》。若實作方向需要偏離建構書，必須先由老闆確認。
>
> **2026-07-02 修訂**：小航審查後經老闆核准補充——①新增「八、聲望系統建構指南」（老闆決定做完整聲望系統）②干治士／艾斯基維任務地點改用官府／商館（不新增教堂設施）③補目標類型（聲望／友好度、船長能力值）、25 位專屬對話缺口、§9 教育檢核、codex 產生器與 UI 排版約定等提醒。

## 一、清查結論

目前夥伴系統已具備：

- 25 位夥伴資料、港口、費用、職位、能力值與圖鑑。
- 章節、主角線、資金、探索點等被動招募門檻。
- 12 位 ★4～5 夥伴的招募完成對話。
- 顏思齊、鄭芝龍兩人的客座離隊資料。

但「專屬任務」尚未真正完成。目前 `questStages` 只是每次打開畫面時，重新檢查玩家當下是否符合條件；沒有任務接取、進行中、完成旗標、任務日誌，也沒有送貨、海戰、跑港、對話或採集等可操作流程。12 位高星夥伴的劇情只會在所有門檻達成後播放，因此屬於「招募完成演出」，不是任務過程。

| 類別 | 人數 | 現況 |
|---|---:|---|
| ★4～5 高星夥伴 | 12 | 有被動條件清單及招募完成對話，但沒有真正可接取的專屬任務 |
| 有條件的 ★3 夥伴 | 7 | 只有章節／資金／探索門檻，多數連招募劇情也沒有 |
| 直接付費加入 | 6 | 其中只有尤蘇夫符合建構書；另外 5 位原定小任務尚未實作 |

## 二、共通缺失

### P0：玩家看不到條件

- `showModal()` 高度固定，條件內容過長時會被裁切，底部又被「知道了」按鈕遮住。
- 畫面只顯示「查看條件」，沒有在夥伴卡片直接顯示下一個明確目標。
- 沒有捲動、分頁或展開式任務內容。
- 沒有在任務日誌、世界地圖或港口資訊中提示夥伴所在地與下一步。

### P0：截圖中的阿迪卡狀態異常

依目前正式資料，阿迪卡應位於萬丹、沒有前置條件、費用 400 兩。玩家持有 1175 兩時，按鈕應顯示「結識 400兩」，不應在巴達維亞顯示「查看條件」。需要實機重現並確認：

- MatesScene 收到的 `portId` 是否正確。
- 候選人列表與按鈕 callback 是否對應同一位夥伴。
- 切換港口、分頁、重建列表後是否殘留舊互動區。
- 瀏覽器是否仍執行舊版開發伺服器或快取內容。

### P1：沒有真正的任務狀態

- `questStages` 沒有 accepted／active／completed 狀態。
- 資金門檻是即時判定；玩家達標後花錢，畫面會重新變成未完成。
- 沒有永久記錄某一步曾經完成。
- 沒有任務失敗、限時過期、取消或重新接取規則。
- 沒有存檔遷移欄位；若新增任務進度，需將存檔由 v17 升版。

### P1：任務名稱與實際玩法不一致

- 「危機」「請託」「試探」「商帳」等名稱看似有事件，但實際只是章節或金錢門檻。
- 探索型任務只檢查既有探索點，沒有接任務後才出現的提示或劇情。
- 12 份高星劇情全部是條件達標後的入隊對話，沒有任務開場與中段。
- 聲望、友好度、累積貿易額、指定海戰、採集、送貨、跑港與對話任務尚未建立。

### P1：聲望／友好度系統不存在

- 遊戲程式目前完全沒有聲望機制（「名聲」只出現在劇情文字裡），但建構書 §5-7 有 5 位夥伴依賴它：沈有容（冒險名聲）、李旦與蘇鳴崗（商人聲望）、劉香（海盜惡名）、理加（新港社友好度）。
- **老闆 2026-07-02 已決定：補上完整聲望系統**，設計指南見「八、聲望系統建構指南」。
- 注意：林斯豪頓「航海術門檻」、何斌「交涉術門檻」**不屬於**聲望系統——可直接沿用既有船長六圍（航海、交涉，存檔 v14 起已實作），列為目標類型「船長能力值門檻」即可，不必新建系統。

### P1：夥伴專屬對話未達建構書規格

- 建構書 §5-7：「每位夥伴各有 5～10 句專屬對話與 1 張人物圖鑑卡」。人物圖鑑卡已齊；但專屬對話目前只有 12 位高星的招募完成對話，25 位的日常專屬對話（5～10 句）皆未實作。
- 建議與第三階段任務文本一併撰寫（任務開場／中段／完成對話可計入句數），低星夥伴另補簡短日常對話。

## 五、建議驗收標準

1. 玩家第一次遇到有條件夥伴時，能在一個畫面內理解「在哪裡、要做什麼、目前做到哪裡」。
2. 每個標示為「專屬任務」的夥伴，至少有一次主動接取及一次明確完成／回報行為。
3. ★4～5 任務至少包含 2 個階段；★5 可使用 3～4 個階段。
4. 任務進度寫入存檔，不因資金或貨物後續變動而倒退。
5. 三條主角線的限定、客座、離隊與不可招募規則符合建構書。
6. 25 位夥伴都有可完成路徑，或清楚標示該主角線不可招募；不得出現永遠卡住但不解釋的按鈕。
7. 招募完成後正確扣除費用、加入船隊、指派職位、解鎖圖鑑並播放對應劇情。

## 八、聲望系統建構指南（2026-07-02 老闆核准新增）

> 老闆決策：**補上完整聲望系統**，不用替代條件簡化。本節為設計指南；實作時若需偏離，先提案問老闆。

### 8-1. 三軌聲望＋部族友好度

| 軌道 | 資料鍵（建議） | 主要來源 | 對應夥伴 |
|---|---|---|---|
| 冒險名聲 | `reputation.adventure` | 確認探索點、發現風景、完成探險委託 | 沈有容 |
| 商人聲望 | `reputation.trade` | 賣出成交額累積、完成採購委託 | 李旦、蘇鳴崗（阿迪卡的「萬丹累積貿易額」可共用同一套成交統計，只是限定萬丹港） |
| 海上威名 | `reputation.valor` | 海戰勝利（依敵方強度加權）、完成海戰委託 | 劉香 |
| 新港社友好度 | `friendship.sinckan` | 西拉雅相關探索（西拉雅平原）、笨港／大員委託、理加任務對話 | 理加 |

- **命名說明（待老闆過目）**：建構書劉香條件原文是「海盜惡名門檻」，但本遊戲主角不當海盜、不做掠奪玩法，故詮釋為「海上威名」——讓海盜圈聽過你的實力，劉香才願意見你。若老闆對名稱或詮釋有意見，改回或另取名皆不影響架構。
- 友好度與聲望分開存放：友好度是「對特定族群／勢力」的關係值，首版只做 `sinckan`（新港社），日後如需其他部族或勢力可直接擴充鍵值。

### 8-2. 資料與存檔

- `state.reputation = { adventure, trade, valor }`（皆從 0 起算；首版只增不減，不做扣減與衰退，降低小學生挫折感）。
- `state.friendship = { sinckan: number }`。
- 與夥伴任務進度（`mateQuests`）**同一次升版**（v17 → v18）並寫遷移，避免連續兩次動存檔結構。
- 舊檔遷移建議：冒險名聲可依「已確認探索點數 × 每點分值」回填，避免老玩家歸零；商人聲望與海上威名若無既有統計可回填則補 0，但門檻要設在中期玩家數次航程內可自然補回的量級，舊玩家不可比新玩家更難達成。
- 數值常數集中在 `state.ts`（比照交易系統 `SAT_*` 常數模式），首版待試玩微調。

### 8-3. 取得事件與首版建議值

| 事件 | 聲望變化（首版建議，實作時可試算調整） |
|---|---|
| 賣出貨物 | 商人聲望 +（成交額 ÷ 1000，向下取整） |
| 完成採購委託 | 商人聲望 +5 |
| 海戰勝利 | 海上威名 +3（敵方 tier 每高一級再 +1） |
| 完成海戰委託 | 海上威名 +5 |
| 確認探索點 | 冒險名聲 +6 |
| 發現風景 | 冒險名聲 +2 |
| 完成探險委託 | 冒險名聲 +5 |
| 西拉雅平原探索、笨港／大員委託完成 | 新港社友好度 +3～6 |
| 理加任務對話節點 | 新港社友好度 +10（任務劇情獎勵） |

- 門檻設定原則：★4～5 夥伴的聲望門檻應落在「玩到該章節的玩家正常遊玩即接近達標、專門衝一下就過」的量級，不設成必須反覆刷的數字。
