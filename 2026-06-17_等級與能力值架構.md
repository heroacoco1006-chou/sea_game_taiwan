# 等級與能力值系統架構（v1 提案）

> 本檔是「等級機制＋主人翁/NPC 能力值」的設計檔。**老闆已於 2026-06-17 確認設計（見 §8）**。
> 這是建構書既有設計之外的新系統；依老闆指示，**先把 M4 收尾，之後再進入實作**，實作時同步更新建構書與 `memory.md`。
> 設計原則延續本專案：給國小五六年級玩、白話、不血腥、與既有海戰／貿易／航海／探索系統對接。

---

## 1. 設計目標

- **成長動機**：玩家從新手船長一路變強，給長線遊玩一個明確的「我在變強」回饋。
- **讓夥伴有意義**：高星／專長夥伴的「能力值」決定他擔任職位時的加成大小，呼應建構書「招募名人＝強力夥伴」的定位。
- **教育不衝突**：能力值是遊戲機制，不竄改史實；圖鑑仍以史實為準。能力高低反映「遊戲中的專長」，非對歷史人物的價值評斷（架空登場者照舊註明真實年代）。

## 2. 六大能力值

| 能力 | 代號 | 主要影響（接入的現有系統） | 對應職位 |
|------|------|--------------------------|----------|
| 統率 | lead | 接舷戰氣勢、戰損緩衝、船員疲勞累積 | 副隊長、水手長 |
| 砲術 | gun | 海戰砲擊威力 | 砲術長 |
| 武勇 | val | 接舷近戰戰力（船長個人，亦受武器裝備影響） | （船長專長，無對應職位） |
| 航海 | nav | 航速、抗暴風、海上事件規避、補給效率 | 航海長 |
| 知識 | kno | 探索發現率、探索疲勞、軍師戰術、醫術 | 軍師、書記、探險嚮導、醫師 |
| 交涉 | neg | 買賣價格、招募條件、情報品質 | 主計長、通譯 |

- **數值範圍**：1～99（仿大航海時代手感；數字直覺、好懂）。
- **顯示位置**：選單→人物資訊頁顯示船長六圍＋等級＋經驗條；船隊資訊頁顯示在隊夥伴六圍。

## 4. 等級與經驗系統

- **等級上限**：50。
- **升級曲線**：升到下一級所需經驗 = `100 + (目前等級−1) × 60`（Lv1→2 需 100，Lv49→50 需 2,980；全程約 7.5 萬經驗）。
- **每級成長**：每升一級獲得 **+5 能力點**，依該主角成長傾向權重分配（四捨五入，餘數補到權重最高項）；每項上限 99。Lv50 約 +245 點，使專長能力接近 80～99、弱項約 40～60。
- **經驗來源（初版，數值可調）**：

  | 行為 | 經驗 |
  |------|------|
  | 完成主線章節 | +400 |
  | 海戰委託完成 | +100 |
  | 探索委託完成 | +80 |
  | 採購委託完成 | +60 |
  | 海戰勝利（隨敵強度） | +40～80 |
  | 新解鎖圖鑑／探索發現 | +30 |
  | 招募夥伴 | +40 |

- **升級表現**：升級時跳出提示（哪些能力提升了多少），不打斷當前流程。

## 6. 能力值如何接入現有系統

> 現有加成多為「有裝備／有職位就 +固定值」。改為「**船長能力＋在隊夥伴對應能力**共同決定加成大小」，裝備與職位效果疊加其上。

| 現有函式 | 接入能力 | 改造方向 |
|----------|----------|----------|
| `boardBonus`（接舷） | 統率＋武勇 | 加上 `(統率+武勇)/常數` 的氣勢/戰力 |
| `cannonMod`（砲擊） | 砲術 | 砲術越高，砲擊倍率越高 |
| `crewSpeedMod`/`gearSpeedMod`（航速） | 航海 | 航海越高，航速加成越高 |
| `stormDamageMod`（抗暴風） | 航海 | 航海越高，暴風損害越低 |
| `fatigueMod`/`reduceCrewLoss`（疲勞／減員） | 統率 | 統率越高，疲勞累積與戰損越低 |
| `tradeBonus`（買賣價） | 交涉 | 交涉越高，買低賣高幅度越大 |
| `explorationFindChance`（探索發現） | 知識 | 知識越高，發現率越高 |

**夥伴職位加成改造**：職位加成 = 擔任該職位夥伴「對應能力值」換算的數值（例：砲術長加成 ∝ 該夥伴砲術）。如此一來，高星專長夥伴明顯比低星強，「指派誰當砲術長」變成有意義的選擇。

## 7. 存檔與相容

- **新增欄位**：`hero.level`、`hero.xp`、`hero.stats{lead,gun,val,nav,kno,neg}`；夥伴能力可即時由星級表算（不必存），或存快取以便個別微調。
- **存檔版本**：升一版（目前 v13 之後）；舊存檔載入時依主角給 Lv1 預設能力、夥伴依星級表補上，不影響既有進度。

## 8. 設計定案（老闆已確認 2026-06-17）

1. **能力數量**：✅ 維持**六圍**（統率／砲術／武勇／航海／知識／交涉）。
2. **夥伴是否升級**：✅ **夥伴固定**（依星級表，不升級）——強化「招募到強力 NPC」的收集目的性。
3. **升級點數分配**：✅ **自動**依成長傾向加點（不開放玩家手動加點）。
4. **數值尺度**：✅ **1～99**。
5. **經驗來源與升級曲線**：✅ 維持本檔 §4 目前版本（初版數值，實作後再視試玩微調）。

---

## 9. 小結

本系統把「等級成長」與「夥伴能力值」綁在一起，讓主線推進、海戰、貿易、探索都回饋到船長變強，也讓 25 位夥伴的招募更有策略性。實作拆解見 `status.md`「等級與能力值系統」區塊。
