# 港町整體美術一致化架構

## 1. 問題定義

目前港町畫面已把各設施換成 V2 精緻圖片，但整體場景仍是 M2 時期的簡單程式繪製：

- 地面是單色矩形，缺少材質、光影與港町氛圍。
- 道路是半透明色塊，和精緻建築的舊紙質感不一致。
- 設施圖片保留「卡片框」感，像浮在地圖上，不像城鎮裡的建築。
- 背景港景只是半透明大圖貼在上方，沒有和街道、碼頭、建築層次融合。
- 樹、民宅、水井、貨箱仍是簡單幾何圖形，和 V2 素材落差大。

結論：這不是一般 UI 美化能解決的問題，應新增一個 M5 子項：**M5-2.5 港町場景整體化**。

## 2. 改造目標

目標是讓玩家進入任何城市時，看到的是同一套 V2 精緻手繪航海 RPG 場景，而不是「精緻設施卡片貼在舊原型地圖上」。

完成後應達到：

- 地面、道路、港口、碼頭、裝飾物、設施建築、角色都屬於同一美術語言。
- 不再使用大面積純色色塊作為主要地面。
- 設施建築不再像卡片，改成透明背景或半透明自然陰影的環境建築。
- 不同文化圈港町仍有差異：漢式、日本、琉球、南洋、歐式。
- 保留現有玩法：走動、點擊移動、進門、港口固定在最下方中央、其他設施配置差異化。

## 3. 場景分層架構

`PortScene` 應改成明確分層，而不是在 `create()` 內直接一段段畫完。

建議層級如下：

| 層級 | 內容 | 深度 |
|------|------|------|
| `base` | 港町底圖、地面材質、遠景色調 | 0 |
| `roads` | 主街、支路、廣場、碼頭連接路 | 1 |
| `waterDock` | 海面、岸線、棧橋、停泊船 | 2 |
| `backProps` | 遠景小屋、倉庫、城牆、樹林 | 3 |
| `buildings` | 可進入設施建築 | 5 |
| `labels` | 設施小牌匾、互動提示標記 | 7 |
| `actor` | 主角、NPC、可動物件 | 10 |
| `frontProps` | 前景箱子、樹、旗幟、欄杆 | 12 |
| `fixedUI` | 上方資訊列、提示文字、小地圖 | 100+ |

技術上可用 Phaser `Container` 或直接用深度管理；重點是不要讓背景、建築、裝飾、UI 混在同一個 graphics 物件裡。

## 4. 素材規格

### 4-1. 港町底圖

新增文化圈底圖或底圖組件：

| 文化圈 | 建議底圖氛圍 |
|--------|--------------|
| `han` | 福建沿海石板路、土黃色牆面、紅瓦、媽祖廟與商館街 |
| `wa` | 日本港町砂土地、町屋街、木格窗、石燈籠 |
| `ryu` | 琉球紅瓦、石垣、海風庭院、首里感 |
| `sea` | 南洋木棧道、椰樹、香料市場、臨水高腳建築 |
| `euro` | 歐式殖民港、石砌廣場、拱廊、倉庫與砲台 |

初期可先做 5 張大底圖，每張建議 2000×1100，對應現有 `TOWN_W` / `TOWN_H`。

長期若效能或彈性需要，再拆成 tiles：地面 512×512、道路 512×512、岸線 512×256、廣場 512×512。

### 4-2. 設施建築

現有 `assets/m5/v2/m5-2/ports/buildings/` 多數是「帶框建築卡片」，適合圖鑑或選單，不適合直接放在走動式港町。

需要新增一組「港町環境建築 cutout」：

- 路徑建議：`assets/m5/v2/m5-2/ports/town-buildings/`
- 格式：PNG，透明背景，建築本體可帶自然投影，但不能有卡片邊框、紙張底色、裝飾框。
- 尺寸：建議源圖 512×384 或 512×512，進遊戲時依設施縮放。
- 每個文化圈至少覆蓋：交易所／酒館／旅館／官府或商館／道具屋／造船廠／港口。

### 4-3. 場景裝飾物

現有樹、水井、貨箱、民宅是程式幾何圖，需替換成 V2 props。

新增路徑建議：`assets/m5/v2/m5-2/ports/props/`

第一批至少：

- 樹：漢式榕樹、日本松、南洋椰樹、歐式庭園樹。
- 小物：貨箱、木桶、井、旗幟、石燈籠、欄杆、碼頭柱、繩索、攤販。
- 民宅：不可進入小屋 2～3 種，可依文化圈混用。

### 4-4. 設施標籤

設施名稱不應再用大塊深色文字蓋住建築中央。建議改為：

- 小牌匾掛在建築門口或屋簷下。
- 滑鼠 hover 或玩家接近時才放大或高亮。
- 遠距離只顯示簡短文字：`商館`、`酒館`、`交易所`。
- 港口可保留 `港口（補給・出航）`，但顯示時可分兩行或用小 icon。

## 5. 程式架構

### 5-1. 新增資料檔

建議新增：

`src/data/portTownThemes.json`

用途：描述文化圈美術主題。

```json
{
  "han": {
    "base": "town_base_han",
    "road": "stone_road_han",
    "dock": "dock_han",
    "props": ["barrel", "cargo_crate", "banyan_tree"],
    "labelStyle": "wood_plaque"
  }
}
```

建議新增：

`src/data/portTownLayouts.json`

用途：把設施位置、道路、裝飾點資料化。現有隨機配置邏輯可先保留，但最後應逐步改成「港口 seed + layout slot + 文化圈 theme」。

```json
{
  "default": {
    "dock": { "x": 1000, "y": 1010 },
    "roads": [
      { "type": "main_horizontal", "x": 1000, "y": 455, "w": 2000, "h": 110 },
      { "type": "main_vertical", "x": 1000, "y": 610, "w": 120, "h": 780 }
    ],
    "facilitySlots": [
      { "x": 320, "y": 285 },
      { "x": 720, "y": 240 }
    ]
  }
}
```

### 5-2. `PortScene` 拆分

現有 `create()` 負責太多事情，建議拆成：

- `createTownBase(theme)`
- `createRoads(layout, theme)`
- `createDock(theme)`
- `createBuildings(layout, theme)`
- `createTownProps(layout, theme)`
- `createPlayer()`
- `createFixedUi()`
- `createMinimap()`

碰撞與互動不要直接依圖片矩形全域判定，應新增每棟建築的：

- `hitbox`：玩家不能穿過的範圍。
- `door`：可進入設施的互動點。
- `labelAnchor`：牌匾顯示位置。

### 5-3. `src/art.ts` 載入擴充

新增 glob：

- `PORT_TOWN_BASE_URLS`
- `PORT_TOWN_BUILDING_URLS`
- `PORT_TOWN_PROP_URLS`
- `PORT_TOWN_ROAD_URLS`

材質 key 命名：

- `portTownBaseKey(id)` → `m5townbase_${id}`
- `portTownBuildingKey(id)` → `m5townb_${id}`
- `portTownPropKey(id)` → `m5townp_${id}`
- `portTownRoadKey(id)` → `m5townr_${id}`

## 6. 分階段導入

### Phase A：架構與資料準備

- 建立本文件。
- 在 `status.md` 增加 M5-2.5 港町場景整體化。
- 明確把「港町整體化」和「M5-6 UI 美化」分開。

### Phase B：快速視覺止血

目標：先讓目前畫面不要突兀。

- 用 V2 港景底圖填滿上半部，不再只用半透明矩形。
- 地面改成低對比舊紙／石板材質，不再是大色塊。
- 道路改為有邊緣、磨損、石板或泥土地紋理的圖層。
- 程式幾何樹、水井、貨箱先替換成簡單 V2 icon/prop。
- 設施卡片縮小或裁掉外框，減少「浮在地圖上」的感覺。

這階段可以先不新增完整 cutout 建築，但要讓整體突兀感下降。

### Phase C：正式 cutout 建築素材

目標：把設施從「卡片」變成「城鎮建築」。

- 用 imagegen/image2.0 生成 5 文化圈 × 7 設施 cutout。
- 寫切片腳本與 manifest。
- `assets/CREDITS.md` 登錄。
- `PortScene` 改用 cutout 建築；舊 building card 保留給圖鑑、選單或 fallback。

### Phase D：資料化 layout 與互動熱區

目標：降低 `PortScene` 硬編碼，讓之後每個港能微調。

- 建立 `portTownThemes.json` 與 `portTownLayouts.json`。
- 導入 `hitbox`、`door`、`labelAnchor`。
- 港口固定中下方，其餘設施仍依港口 seed 變化，但不再互卡。
- 小地圖改讀 layout，不自己猜建築座標。

### Phase E：完整驗收

至少檢查：

- 大員或澎湖：漢式／台灣港。
- 月港或泉州：漢式商港。
- 平戶或長崎：日本港。
- 那霸：琉球港。
- 巴達維亞或馬尼拉：歐式／南洋港。

## 7. 驗收標準

完成 M5-2.5 後，應符合：

- 看不到大面積原型期純色地面。
- 看不到設施圖片像卡片浮在場景上。
- 建築、地面、道路、裝飾、角色在同一視覺風格中。
- 玩家仍可清楚知道哪裡可進入，不因美術整合而找不到設施。
- 港口固定中下方，補給與出航位置直覺。
- 每個文化圈港町有明顯差異，但操作邏輯一致。
- `npm run build` 通過。
- 瀏覽器以 `http://127.0.0.1:5173/` 測試，console 無錯誤。

## 8. 風險與注意事項

- **素材體積**：若每個文化圈都用 2000×1100 大底圖，bundle 會變大；M5-8 前可改 lazy load。
- **碰撞判定**：建築 cutout 不是矩形，仍需手動設 hitbox，不可直接用圖片尺寸全擋。
- **文字可讀性**：牌匾不能太小；小朋友仍要一眼看出功能。
- **探索感**：設施位置可變，但不能變成找不到入口；港口與旅館等核心設施應有視覺引導。
- **舊素材去向**：現有帶框 building card 不刪除，可轉作圖鑑、設施選單或 fallback。
