# 世界地圖整體美術一致化架構

## 1. 問題定義

目前大地圖已接入 M5-2 的 V2 復古海圖底圖，但 `WorldMapScene` 仍把 `map.json` 的陸地多邊形用單色塊畫在上層，造成視覺斷裂：

- 海面有舊海圖質感，但陸地是平面色塊，看起來像後貼的原型圖。
- 海岸線是直線折角，多邊形節點明顯，尤其日本、韓國、台灣、琉球與東南亞群島更突兀。
- 港口與探索點雖已修正可達性，但視覺上仍可能和海岸線、陸地輪廓不自然。
- 小地圖也沿用同一批簡化多邊形，縮小後更像方塊示意圖。

結論：這和港町一樣不是單純 UI 美化，應新增 **M5-2.6 世界地圖場景整體化**，專門處理海圖、陸地、海岸線、地名、港口、探索點與碰撞資料之間的一致性。

## 2. 核心原則

### 2-1. 玩法資料不可被美術圖直接取代

目前 `src/data/map.json` 同時承擔：

- 陸地碰撞判定。
- 港口、探索點與風景附近的可達性基準。
- 小地圖陸地形狀。

因此不能直接把一張精緻世界地圖設成背景，然後讓玩家看圖航行；如果背景圖和 `map.json` 不一致，會再次出現「看得到卻進不去」或「看起來在海上其實撞到陸地」的問題。

短期策略：**以 `map.json` 為互動權威，改良它的視覺呈現。**

中長期策略：**用 `map.json` 產生精緻海圖 source，或從精緻 source 回填校正後的 vector/hitbox。** 不讓視覺圖和碰撞資料各自漂移。

### 2-2. 大地圖分層

建議 `WorldMapScene` 拆成以下視覺層：

| 層級 | 內容 | 深度 |
|------|------|------|
| `seaBase` | 復古海圖底色、紙張紋理、海域色階 | 0 |
| `seaDetails` | 波紋、洋流、航線淡線、羅盤玫瑰裝飾 | 1 |
| `landBase` | 陸地填色、山脈／森林／平原大區塊 | 2 |
| `coastline` | 海岸描邊、淺灘、島嶼邊緣、破碎岸線 | 3 |
| `labels` | 地區名、海域名、文化圈字樣 | 4 |
| `markers` | 港口、探索點、風景、海盜、任務標記 | 6-9 |
| `ship` | 玩家船、NPC 船、任務船 | 10 |
| `fixedUI` | HUD、羅盤、小地圖、提示文字 | 100+ |

## 3. 推薦實作路線

### Phase A：架構與追蹤

- 建立本文件。
- 在 `status.md` 新增 M5-2.6 世界地圖整體化。
- 明確註記：M5-2 第一輪只是接入海圖底圖與圖示，尚未完成陸地整體化。

### Phase B：快速視覺止血（不改座標資料）

目標：不動 `map.json` 互動資料，只讓目前陸地不再像方塊。

做法：

- `WorldMapScene` 新增 `createSeaBase()` 與 `createLandVisuals()` 分層；標籤、港口、探索點與船隻維持既有建立流程，但深度要接在陸地與海岸層之後。
- 陸地多邊形保留，但改成多層繪製：底色、內部紋理、海岸外描邊、淺灘光暈。
- 在多邊形內用 deterministic noise 加入山脈、森林、平原斑塊，但不要影響碰撞。
- 海岸線用較粗的深色線 + 淺色外緣線，降低直線折角的生硬感。
- 島嶼可加「周圍淺灘橢圓／破碎浪線」，讓琉球、澎湖、香料群島不只是小多邊形。
- 小地圖同步使用改良色盤，但保持簡化，避免太雜。

此階段不新增大型圖檔，風險最低，可快速改善突兀感。

### Phase B 導入紀錄（2026-06-21）

- `WorldMapScene` 已拆出 `createSeaBase()` 與 `createLandVisuals()`，把海面、陸地、海岸、標籤與標記拉開深度層級。
- 海面保留 V2 `sea_chart`，提高紙海圖底圖存在感，並加上海色疊層、低透明波紋與航線／洋流淡線。
- 陸地仍以 `map.json` 多邊形為互動權威，但視覺上改成底色、內部斑塊紋理、山形 relief、深色海岸線、淺色海岸外緣與淺灘光暈。
- 島嶼與海岸邊新增 deterministic 的浪線裝飾，讓澎湖、琉球、日本、東南亞島嶼不再只有單色折線邊界。
- 小地圖同步改用新陸地色盤並加描邊，但仍保持簡化，不在小地圖塞入過多紋理。
- 本階段未改港口、探索點、碰撞與互動半徑；可達性仍沿用既有 `map.json` 與先前校正結果。

### Phase C：精緻大地圖 source 生成

目標：建立一張真正符合本遊戲座標的 V2 精緻大海圖，不再只是把泛用 `sea_chart` 當透明底圖。

建議方法：

1. 寫腳本從 `map.json` 輸出一張高解析黑白／色塊 land-mask source。
2. 用 imagegen/image2.0 以 land-mask 作構圖約束，生成東亞到東南亞的 17 世紀復古航海圖。
3. 生成後不要直接改座標；先用人工比對確認台灣、澎湖、琉球、日本、朝鮮、呂宋、爪哇、香料群島位置是否貼合。
4. 若偏差過大，回到 land-mask 重出，而不是把港口硬拖去配合錯圖。

輸出建議：

- `assets/m5/v2/m5-2/world/full_map_v2.png`：世界地圖完整底圖。
- `assets/m5/v2/m5-2/world/full_map_v2_mask.png`：與 `map.json` 對齊的陸地遮罩／驗證圖。
- `assets/m5/v2/m5-2/world/full_map_v2_notes.md`：生成 prompt、比對紀錄、偏差清單。

### Phase D：資料校正與可達性驗收

目標：視覺、碰撞與港口位置三者一致。

- 檢查每個港口是否在海岸附近且可進港。
- 檢查探索點是否不在不合理海面上；內陸點可透過「近岸入口」表示，不直接把城市圖示放海上。
- 若有必要，微調 `map.json` 的陸地多邊形或港口座標，但每次改動都要測：
  - 港口可進入。
  - 航道不被陸地封死。
  - 探索點提示距離正常。
  - 小地圖能反映變更。

### Phase E：最終整合

- `WorldMapScene` 改用 full map 作主要視覺底圖。
- 陸地多邊形可改成低透明碰撞輔助層，或只保留 coastline / hover debug 模式。
- 小地圖可由同一份 land-mask 縮圖產生，避免和主畫面不一致。
- M5-8 前處理 lazy load 或壓縮，避免世界圖與既有素材讓 bundle 過大。

## 4. 驗收地點

至少檢查以下區域：

- 台灣／澎湖／大員：港口、海岸、探索點與主線核心位置必須自然。
- 福建沿海：月港、泉州、福州、安海不可離岸過遠或卡陸地。
- 琉球：那霸可進入，首里城周邊探索點不應壓港口。
- 日本：平戶、長崎、堺與雲仙山路要視覺合理、可達。
- 朝鮮：漢陽／首爾類探索點應以近岸入口或內陸標示呈現，不能像城市直接漂在海上。
- 東南亞：馬尼拉、巴達維亞、萬丹、安汶、德那第、班達群島要兼顧航道與島嶼辨識。

## 5. 風險與注意事項

- **不要讓美術圖破壞可玩性**：港口和探索點可達性優先於畫面好看。
- **imagegen 不擅長精準地理座標**：必須用 `map.json` 或 mask 約束構圖，不能只靠文字 prompt。
- **碰撞資料要可追蹤**：若修改 `map.json`，需同步記錄原因，避免日後不知為何移動某個海岸點。
- **小朋友閱讀性**：地名與港口標籤不能被過度紋理淹沒。
- **效能**：世界地圖尺寸大，若使用 full map PNG，要評估檔案大小與低階機載入速度。
